# Algorithm Definition

## 1. ルールA: カット処理（Cut Algorithm）

### 1.1 Overview
- algorithm_id: ALG-001
- description: 店舗什器幅が標準棚割幅より小さい場合に、売上ランクの低い商品から順にフェイス数を削減・商品を削除し、棚幅に収まるよう調整する

### 1.2 Input
| name | type | description |
|------|------|-------------|
| standardProducts | StandardPlanogramProduct[] | 標準棚割の商品配置一覧 |
| storeWidth | number | 店舗什器の幅（mm） |
| standardWidth | number | 標準棚割の幅（mm） |
| products | Product[] | 商品マスタ（売上ランク参照用） |

### 1.3 Output
| name | type | description |
|------|------|-------------|
| storeProducts | StorePlanogramProduct[] | カット適用後の商品配置一覧 |
| warnings | string[] | カット対象商品の警告メッセージ |

### 1.4 Logic
```pseudo
FUNCTION applyCutRule(standardProducts, storeWidth, products):
    storeProducts = COPY(standardProducts)
    warnings = []

    FOR EACH shelf IN unique(storeProducts.shelfIndex):
        shelfProducts = FILTER(storeProducts, p => p.shelfIndex == shelf)

        // 売上ランク降順（ランク値が大きい＝売上が低い商品を先に処理）
        SORT shelfProducts BY salesRank DESC

        totalWidth = SUM(p.width * p.faceCount FOR p IN shelfProducts)

        WHILE totalWidth > storeWidth:
            target = shelfProducts[LAST]  // 最も売上ランクが低い商品

            IF target.faceCount > 1 THEN
                target.faceCount = target.faceCount - 1
                target.isCut = true
                ADD WARNING "商品[{name}]のフェイス数を削減"
            ELSE
                REMOVE target FROM shelfProducts
                target.isCut = true
                ADD WARNING "商品[{name}]をカット"
            END IF

            totalWidth = SUM(p.width * p.faceCount FOR p IN shelfProducts)
            RE-SORT shelfProducts BY salesRank DESC
        END WHILE

        // 残存商品を左詰めで再配置
        currentX = 0
        FOR EACH p IN shelfProducts ORDER BY positionX:
            p.positionX = currentX
            currentX = currentX + (p.width * p.faceCount)
        END FOR
    END FOR

    MARK ALL storeProducts AS isAutoGenerated = true
    RETURN (storeProducts, warnings)
END FUNCTION
```

## 2. ルールB: 拡張処理（Expand Algorithm）

### 2.1 Overview
- algorithm_id: ALG-002
- description: 店舗什器幅が標準棚割幅より大きい場合に、売上上位商品のフェイス数を拡大して棚スペースを充填する

### 2.2 Input
| name | type | description |
|------|------|-------------|
| standardProducts | StandardPlanogramProduct[] | 標準棚割の商品配置一覧 |
| storeWidth | number | 店舗什器の幅（mm） |
| standardWidth | number | 標準棚割の幅（mm） |
| products | Product[] | 商品マスタ（売上ランク参照用） |

### 2.3 Output
| name | type | description |
|------|------|-------------|
| storeProducts | StorePlanogramProduct[] | 拡張適用後の商品配置一覧 |
| warnings | string[] | 警告メッセージ |

### 2.4 Logic
```pseudo
CONSTANTS:
    EXPAND_TOP_RANK_LIMIT = 10
    EXPAND_MULTIPLIER = 2.0
    EXPAND_SUB_MULTIPLIER = 1.5

FUNCTION applyExpandRule(standardProducts, storeWidth, products):
    storeProducts = COPY(standardProducts)
    warnings = []

    FOR EACH shelf IN unique(storeProducts.shelfIndex):
        shelfProducts = FILTER(storeProducts, p => p.shelfIndex == shelf)

        // 売上ランク昇順（ランク値が小さい＝売上が高い商品を先に処理）
        SORT shelfProducts BY salesRank ASC

        // 第1パス: 上位10商品のフェイス数を2倍
        originalFaceCounts = SAVE(shelfProducts.faceCount)

        FOR i = 0 TO MIN(EXPAND_TOP_RANK_LIMIT, LENGTH(shelfProducts)) - 1:
            shelfProducts[i].faceCount = FLOOR(shelfProducts[i].faceCount * EXPAND_MULTIPLIER)
        END FOR

        totalWidth = SUM(p.width * p.faceCount FOR p IN shelfProducts)

        IF totalWidth > storeWidth THEN
            RESTORE shelfProducts.faceCount FROM originalFaceCounts
        ELSE
            // 第2パス: 11位以降のフェイス数を1.5倍
            secondOriginal = SAVE(shelfProducts.faceCount)

            FOR i = EXPAND_TOP_RANK_LIMIT TO LENGTH(shelfProducts) - 1:
                shelfProducts[i].faceCount = FLOOR(shelfProducts[i].faceCount * EXPAND_SUB_MULTIPLIER)
            END FOR

            totalWidth = SUM(p.width * p.faceCount FOR p IN shelfProducts)

            IF totalWidth > storeWidth THEN
                RESTORE shelfProducts.faceCount FROM secondOriginal
            END IF
        END IF

        // 残りスペースは空白のまま（余白として残す）
        currentX = 0
        FOR EACH p IN shelfProducts ORDER BY positionX:
            p.positionX = currentX
            currentX = currentX + (p.width * p.faceCount)
        END FOR
    END FOR

    MARK ALL storeProducts AS isAutoGenerated = true
    RETURN (storeProducts, warnings)
END FUNCTION
```

## 3. ルールC: 同期処理（Sync Algorithm）

### 3.1 Overview
- algorithm_id: ALG-003
- description: 標準棚割の最新状態を取得し、個店棚割にルールA/Bを再適用して同期する

### 3.2 Input
| name | type | description |
|------|------|-------------|
| storePlanogram | StorePlanogram | 既存の個店棚割 |
| standardPlanogram | StandardPlanogram | 最新の標準棚割 |
| storeWidth | number | 店舗什器の幅（mm） |

### 3.3 Output
| name | type | description |
|------|------|-------------|
| updatedStorePlanogram | StorePlanogram | 同期後の個店棚割 |

### 3.4 Logic
```pseudo
FUNCTION syncStorePlanogram(storePlanogram, standardPlanogram, storeWidth):
    standardWidth = standardPlanogram.width

    IF storeWidth < standardWidth THEN
        (products, warnings) = applyCutRule(standardPlanogram.products, storeWidth, productMaster)
    ELSE IF storeWidth > standardWidth THEN
        (products, warnings) = applyExpandRule(standardPlanogram.products, storeWidth, productMaster)
    ELSE
        products = COPY(standardPlanogram.products)
        warnings = []
    END IF

    storePlanogram.products = products
    storePlanogram.warnings = warnings
    storePlanogram.status = 'synced'
    storePlanogram.syncedAt = NOW()

    SAVE(storePlanogram)
    RETURN storePlanogram
END FUNCTION
```

## 4. ルールD: 商品連結処理（Product Consolidation Algorithm）

### 4.1 Overview
- algorithm_id: ALG-004
- description: 同一段内に同一商品が隣接配置された場合、フェイス数を統合する

### 4.2 Input
| name | type | description |
|------|------|-------------|
| existingProduct | StorePlanogramProduct | 既存の配置済み商品 |
| droppedProduct | Product | ドロップされた商品 |
| faceCount | number | ドロップ時のフェイス数 |

### 4.3 Output
| name | type | description |
|------|------|-------------|
| mergedProduct | StorePlanogramProduct | 統合後の商品配置 |

### 4.4 Logic
```pseudo
FUNCTION consolidateProduct(existingProduct, droppedProduct, faceCount):
    IF existingProduct.productId == droppedProduct.id THEN
        existingProduct.faceCount = existingProduct.faceCount + faceCount
        RETURN existingProduct  // 統合
    ELSE
        RETURN CREATE_NEW_PLACEMENT(droppedProduct, faceCount)  // 新規配置
    END IF
END FUNCTION
```

## 5. AABB衝突検知アルゴリズム（Collision Detection）

### 5.1 Overview
- algorithm_id: ALG-005
- description: 店舗フロア上の什器配置において、Axis-Aligned Bounding Boxによる重なり検知を行う

### 5.2 Input
| name | type | description |
|------|------|-------------|
| rect1 | {x, y, width, height} | 什器1の矩形領域 |
| rect2 | {x, y, width, height} | 什器2の矩形領域 |

### 5.3 Output
| name | type | description |
|------|------|-------------|
| isColliding | boolean | 衝突している場合true |

### 5.4 Logic
```pseudo
FUNCTION checkAABBCollision(rect1, rect2):
    RETURN (
        rect1.x < rect2.x + rect2.width AND
        rect1.x + rect1.width > rect2.x AND
        rect1.y < rect2.y + rect2.height AND
        rect1.y + rect1.height > rect2.y
    )
END FUNCTION
```

## 6. 売上ランク計算アルゴリズム（Sales Rank Calculation）

### 6.1 Overview
- algorithm_id: ALG-006
- description: 商品の販売数量から売上ランク（1-100）を自動計算する

### 6.2 Input
| name | type | description |
|------|------|-------------|
| salesQuantity | number | 販売数量 |
| allProducts | Product[] | 全商品リスト（相対ランク計算用） |

### 6.3 Output
| name | type | description |
|------|------|-------------|
| salesRank | number | 売上ランク（1-100、1が最高） |

### 6.4 Logic
```pseudo
FUNCTION calculateSalesRank(salesQuantity, allProducts):
    // 全商品を販売数量で降順ソート
    sorted = SORT(allProducts BY salesQuantity DESC)

    // パーセンタイルベースのランク計算
    position = FIND_INDEX(sorted, salesQuantity)
    percentile = position / LENGTH(sorted)
    rank = FLOOR(percentile * 99) + 1  // 1-100の範囲

    RETURN CLAMP(rank, 1, 100)
END FUNCTION
```

## 7. ヒートマップ色計算アルゴリズム（Heatmap Color Calculation）

### 7.1 Overview
- algorithm_id: ALG-007
- description: メトリクス値を5段階の色スケール（赤→橙→黄→緑→青）にマッピングする

### 7.2 Input
| name | type | description |
|------|------|-------------|
| value | number | メトリクス値 |
| min | number | データセット内の最小値 |
| max | number | データセット内の最大値 |

### 7.3 Output
| name | type | description |
|------|------|-------------|
| color | string | CSS色コード |

### 7.4 Logic
```pseudo
FUNCTION getHeatmapColor(value, min, max):
    IF max == min THEN
        RETURN YELLOW  // 全値同一の場合
    END IF

    ratio = (value - min) / (max - min)  // 0.0 - 1.0

    IF ratio >= 0.8 THEN RETURN RED       // 上位20%
    ELSE IF ratio >= 0.6 THEN RETURN ORANGE  // 上位40%
    ELSE IF ratio >= 0.4 THEN RETURN YELLOW  // 中位
    ELSE IF ratio >= 0.2 THEN RETURN GREEN   // 下位40%
    ELSE RETURN BLUE                          // 下位20%
    END IF
END FUNCTION
```

## 8. 一括生成バッチ処理アルゴリズム（Batch Generation）

### 8.1 Overview
- algorithm_id: ALG-008
- description: 複数店舗に対して標準棚割からの個店棚割生成を逐次実行し、進捗を通知する

### 8.2 Input
| name | type | description |
|------|------|-------------|
| standardPlanogram | StandardPlanogram | 元となる標準棚割 |
| storeIds | string[] | 対象店舗IDリスト |
| onProgress | callback | 進捗通知コールバック |

### 8.3 Output
| name | type | description |
|------|------|-------------|
| results | StorePlanogram[] | 生成結果一覧 |
| summary | {total, success, warning, error} | 結果サマリ |

### 8.4 Logic
```pseudo
FUNCTION batchGenerateStorePlanograms(standardPlanogram, storeIds, onProgress):
    results = []
    summary = {total: LENGTH(storeIds), success: 0, warning: 0, error: 0}

    FOR i = 0 TO LENGTH(storeIds) - 1:
        storeId = storeIds[i]
        onProgress({current: i + 1, total: summary.total})

        TRY:
            store = GET_STORE(storeId)
            fixtures = GET_STORE_FIXTURES(storeId)
            storeWidth = GET_FIXTURE_WIDTH(fixtures, standardPlanogram.fixtureType)

            IF storeWidth IS NULL THEN
                THROW "什器データ未設定"
            END IF

            standardWidth = standardPlanogram.width

            IF storeWidth < standardWidth THEN
                (products, warnings) = applyCutRule(...)
            ELSE IF storeWidth > standardWidth THEN
                (products, warnings) = applyExpandRule(...)
            ELSE
                products = COPY(standardPlanogram.products)
                warnings = []
            END IF

            status = IF LENGTH(warnings) > 0 THEN 'warning' ELSE 'generated'
            storePlanogram = CREATE_STORE_PLANOGRAM(storeId, products, status, warnings)

            results.ADD(storePlanogram)
            IF status == 'warning' THEN summary.warning++ ELSE summary.success++

        CATCH error:
            storePlanogram = CREATE_STORE_PLANOGRAM(storeId, [], 'error', [error.message])
            results.ADD(storePlanogram)
            summary.error++
        END TRY
    END FOR

    RETURN (results, summary)
END FUNCTION
```

## 9. 階層横断曖昧検索アルゴリズム（Cross-Hierarchy Fuzzy Search）

### 9.1 Overview
- algorithm_id: ALG-009
- description: 商品階層マスタを全8レベル（16フィールド）横断で曖昧検索し、関連度順にソートする

### 9.2 Input
| name | type | description |
|------|------|-------------|
| query | string | 検索クエリ（スペース区切りで複数ワードAND検索） |
| hierarchies | HierarchyEntry[] | 商品階層マスタ全件 |

### 9.3 Output
| name | type | description |
|------|------|-------------|
| results | {entry: HierarchyEntry, score: number}[] | 関連度スコア付きの検索結果（スコア降順） |

### 9.4 Logic
```pseudo
FUNCTION searchHierarchyAcrossLevels(query, hierarchies):
    words = SPLIT(query, /\s+/)  // スペース区切りでワード分割
    results = []

    FOR EACH entry IN hierarchies:
        // 全16フィールド（8レベル × コード＋名称）を検索対象
        fields = [entry.divisionCode, entry.divisionName, ..., entry.subSegmentCode, entry.subSegmentName]

        // 全ワードがいずれかのフィールドに含まれるかチェック（AND条件）
        allWordsMatch = TRUE
        score = 0

        FOR EACH word IN words:
            wordFound = FALSE
            FOR EACH field IN fields:
                IF field CONTAINS word (case-insensitive) THEN
                    wordFound = TRUE
                    score = score + 1  // 一致フィールド数でスコアリング
                END IF
            END FOR
            IF NOT wordFound THEN
                allWordsMatch = FALSE
                BREAK
            END IF
        END FOR

        IF allWordsMatch THEN
            results.ADD({entry, score})
        END IF
    END FOR

    SORT results BY score DESC
    RETURN results
END FUNCTION
```
