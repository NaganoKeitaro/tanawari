## Scenario Detail

### TS-001
- objective: 標準棚割を基に複数店舗の個店棚割が正しく一括生成されることを確認する
- related_usecases: [UC-013, UC-015]
- flow: 標準棚割作成 → 対象店舗選択 → 一括生成実行 → 結果確認

#### steps:
1. FMT=MEGA、什器タイプ=multi-tier、幅=1200mm、段数=5の標準棚割を作成する
2. ブロックをキャンバスに配置し、商品を展開する
3. 標準棚割を保存する
4. 個店棚割バッチ画面でFMT=MEGAを選択する
5. 作成した標準棚割を選択する
6. 3店舗（什器幅: 900mm, 1200mm, 1500mm）を選択する
7. 「一括生成」を実行する
8. 進捗バーが0%→100%に進行することを確認する
9. 結果レポートで3件すべてのステータスを確認する

#### success_criteria:
- 什器幅900mmの店舗: ステータス=warning、ルールAが適用される
- 什器幅1200mmの店舗: ステータス=generated、商品がそのまま配置される
- 什器幅1500mmの店舗: ステータス=generated、ルールBが適用される
- 進捗バーが正常に進行する
- 結果レポートの件数が正しい

### TS-002
- objective: 店舗什器幅が標準幅より小さい場合に、売上ランクの低い商品から正しくカットされることを確認する
- related_usecases: [UC-015]
- flow: 標準棚割準備 → カット条件の店舗で生成 → カット結果検証

#### steps:
1. 標準棚割（幅=1200mm、段数=3）を作成する
2. 各段に売上ランク1-10の商品を配置する（各フェイス数=2、商品幅=100mm）
3. 什器幅=800mmの店舗を対象に個店棚割を生成する
4. 生成結果の各段の商品を確認する
5. カットされた商品の売上ランクを確認する
6. 残存商品のpositionXが左詰めで再計算されていることを確認する

#### success_criteria:
- 各段の商品幅合計が800mm以内に収まる
- 売上ランクの低い商品（ランク値が大きい）から優先的にカットされる
- フェイス数2の商品は先にフェイス数1に削減される
- フェイス数1の商品がカットされた場合、isCut=trueが設定される
- 全商品にisAutoGenerated=trueが設定される
- 警告メッセージにカット対象商品名が含まれる

### TS-003
- objective: 店舗什器幅が標準幅より大きい場合に、売上上位商品のフェイス数が正しく拡張されることを確認する
- related_usecases: [UC-015]
- flow: 標準棚割準備 → 拡張条件の店舗で生成 → 拡張結果検証

#### steps:
1. 標準棚割（幅=900mm、段数=3）を作成する
2. 各段に売上ランク1-15の商品を配置する（各フェイス数=1、商品幅=50mm）
3. 什器幅=1500mmの店舗を対象に個店棚割を生成する
4. 売上ランク上位10商品のフェイス数を確認する
5. 売上ランク11位以降の商品のフェイス数を確認する
6. 各段の商品幅合計が什器幅を超えていないことを確認する

#### success_criteria:
- 売上ランク上位10商品のフェイス数が2倍になる（EXPAND_MULTIPLIER=2.0）
- 11位以降の商品のフェイス数が1.5倍になる（EXPAND_SUB_MULTIPLIER=1.5）
- 各段の合計幅が1500mmを超えない
- 超過する場合はリバートされる
- 全商品にisAutoGenerated=trueが設定される

### TS-004
- objective: 標準棚割を更新した後、個店棚割を同期すると最新状態が反映されることを確認する
- related_usecases: [UC-013, UC-015, UC-017]
- flow: 標準棚割作成 → 個店棚割生成 → 標準棚割更新 → 同期実行 → 結果確認

#### steps:
1. 標準棚割を作成し、商品A/B/Cを配置する
2. 個店棚割を生成する（ステータス=generated）
3. 標準棚割を編集し、商品Dを追加・商品Aを削除する
4. 個店棚割エディタで「同期」ボタンを押下する
5. 同期後の個店棚割の商品リストを確認する
6. ステータスとsyncedAtを確認する

#### success_criteria:
- 同期後、商品Aが削除され、商品Dが追加されている
- ステータスが'synced'に更新される
- syncedAtに同期日時が記録される
- ルールA/Bが正しく再適用される

### TS-005
- objective: Excel/CSVから商品データを正しくインポートできることを確認する
- related_usecases: [UC-004]
- flow: ファイル選択 → プレビュー → 重複処理選択 → インポート → 結果確認

#### steps:
1. 10件の商品を含むExcelファイルを準備する（うち2件はJAN重複）
2. 商品マスタ画面で「インポート」ボタンを押下する
3. ファイルを選択する
4. プレビューで10件が表示されることを確認する
5. 重複処理を「スキップ」に設定してインポートを実行する
6. インポート結果を確認する（8件成功、2件スキップ）
7. 重複処理を「上書き」に変更して再度インポートする
8. 10件すべてが成功することを確認する

#### success_criteria:
- プレビューが正しく表示される
- スキップモードで重複JAN商品がスキップされる
- 上書きモードで重複JAN商品が更新される
- 売上ランクが自動計算される
- 8階層の分類データが正しくマッピングされる
- インポート結果の件数が正確

### TS-006
- objective: 什器の衝突が正しく検知されることを確認する
- related_usecases: [UC-018]
- flow: 店舗選択 → 什器配置 → 衝突検知 → 回避操作

#### steps:
1. 店舗什器配置画面を開く
2. 什器A（幅=1200mm、奥行き=600mm）をposition(0, 0)に配置する
3. 什器B（幅=1200mm、奥行き=600mm）をposition(1000, 0)に配置しようとする
4. 衝突が検知されることを確認する（200mm重なり）
5. 什器Bをposition(1200, 0)に移動する
6. 衝突が解消されることを確認する
7. 什器Aを90度回転させる
8. 回転後の什器Aが什器Bと衝突しないことを確認する

#### success_criteria:
- AABB方式で什器の重なりが正しく検知される
- 衝突時に配置がブロックされる
- 回転後も衝突検知が正しく動作する
- 衝突がない場合は正常に配置できる

### TS-007
- objective: 自動生成された個店棚割を手動で正しく調整できることを確認する
- related_usecases: [UC-016]
- flow: 個店棚割表示 → 商品追加 → フェイス数変更 → 商品削除 → オーバーフロー検知

#### steps:
1. 自動生成済みの個店棚割（幅=1200mm）を開く
2. 商品パレットから新しい商品をドラッグ&ドロップで段1に追加する
3. 配置された商品のフェイス数を+ボタンで増加させる
4. 棚幅オーバーフローが発生するまでフェイス数を増加させる
5. オーバーフローアラートの表示を確認する
6. 既存商品の-ボタンでフェイス数を1まで減少させる
7. 商品を削除する
8. 同一商品を同一段に隣接配置し、フェイス数が統合されることを確認する

#### success_criteria:
- ドラッグ&ドロップで商品が正しく配置される
- フェイス数の増減が正しく動作する
- 棚幅オーバーフロー時にアラートが表示される
- 残りスペースがmm単位で表示される
- 商品削除が正しく動作する
- 同一商品の隣接配置でフェイス数が統合される（ルールD）

### TS-008
- objective: マスタ登録から棚割指示書出力までのエンドツーエンドフローが正常に動作することを確認する
- related_usecases: [UC-001, UC-007, UC-008, UC-011, UC-012, UC-013, UC-015, UC-018, UC-021]
- flow: マスタ登録 → ブロック作成 → 標準棚割設計 → 什器配置 → 個店棚割生成 → 指示書出力

#### steps:
1. 商品マスタに10件の商品を登録する
2. 店舗マスタに2店舗（FMT=MEGA）を登録する
3. 什器マスタに多段什器を登録する
4. 棚ブロックを2つ作成し、各ブロックに5商品を配置する
5. 標準棚割を作成し、2ブロックを配置する
6. 各店舗に什器を配置する
7. 標準棚割から2店舗の個店棚割を一括生成する
8. 生成結果のステータスを確認する
9. 棚割指示書画面で店舗を選択する
10. 注記を入力する
11. プレビューを確認する
12. 印刷を実行する

#### success_criteria:
- 全ステップが正常に完了する
- 個店棚割が正しく生成される
- 棚割指示書にフロアレイアウト・什器別棚割・商品一覧が含まれる
- 指示書の印刷プレビューが正しく表示される
