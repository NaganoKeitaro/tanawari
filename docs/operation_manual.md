## 1. はじめに

### システムの目的

本システムは、小売店舗の棚割（ゴンドラや平台冷蔵ケースなどの什器に、どの商品を・何フェイスで・どの段に配置するか）を効率的に管理するためのWebアプリケーションです。

### 基本的な業務フロー

```
① FMT標準棚割を作成
    ↓
② 個店棚割を自動生成（標準棚割をベースに各店舗の棚サイズに合わせて自動調整）
    ↓
③ 必要に応じて個店ごとに微調整
```

## 2. 基本用語の解説

| 用語 | 説明 |
|------|------|
| **FMT** | 店舗フォーマット。店舗の業態分類のこと。本システムでは **MEGA**、**SuC**、**SMART**、**GO**、**FC** の5種類が定義されています。 |
| **什器（しゅうき）** | 店舗に設置する棚や陳列ケースのこと。「多段棚」「ゴンドラ」「平台冷蔵」「平台冷凍」「平台冷蔵エンド」「平台冷凍エンド」の種類があります。 |
| **尺（しゃく）** | 什器の幅を表す単位。1尺 = 300mm（30cm）です。例: 「3尺棚」= 幅900mmの棚。 |
| **棚ブロック** | 複数の商品をひとまとめにしたグループ単位。ブロック単位でドラッグ＆ドロップして棚割を組み立てます。 |
| **フェイス数** | 棚の正面から見た同一商品の並び数。フェイス数が多いほど、その商品の陳列面が広くなります。 |
| **段（だん）** | 什器の棚板の位置。画面上では **最下部が1段目**、上に行くほど段数が増えます（実際の什器と同じ向き）。 |
| **標準棚割** | FMTごとに定めた「お手本」となる棚割パターン。基準店舗の什器構成をもとに作成します。 |
| **個店棚割** | 各店舗固有の棚割。標準棚割をベースに、店舗の棚幅に合わせて自動調整されたものです。 |
| **リージョン** | 地域を表すグループ（北海道、東北、関東、中部、近畿、中国・四国、九州、全地域）。 |
| **売上ランク** | 商品ごとの売上順位。1が最高ランクです。自動調整時にどの商品を優先するかの判断に使用されます。 |

## 4. FMT標準棚割管理

> **サイドバー** → 📋 **FMT標準棚割**

### この画面でできること

- FMTごとの標準的な棚割パターンを作成・編集する
- 棚ブロックをドラッグ＆ドロップで配置する
- 既存の棚割を複製して新しいパターンを作る
- 分析モードで売上・粗利などのヒートマップを確認する

### 操作手順

#### ステップ1：FMTと基準店舗を選択する

1. 画面上部の「**FMT選択**」ドロップダウンから対象のFMT（MEGA、SuC など）を選択します。
2. 「**基準店舗（什器配置済み）**」ドロップダウンから店舗を選択します。
   - ここに表示されるのは、「店舗棚尺マスタ」で什器を配置済みの店舗のみです。

> ⚠️ 店舗が表示されない場合は、先に「📐 店舗棚尺マスタ」画面で什器を配置してください。

[ここにFMT選択・基準店舗選択エリアのスクリーンショットを挿入]

#### ステップ2：什器タイプを選択する

FMTの下に表示されるタブで什器の種類を切り替えます。

| タブ名 | 対象什器 |
|--------|---------|
| 多段   | 多段棚・ゴンドラ |
| 平台冷蔵 | 平台冷蔵ケース |
| 平台冷蔵エンド | 平台冷蔵エンドケース |
| 平台冷凍 | 平台冷凍ケース |
| 平台冷凍エンド | 平台冷凍エンドケース |

#### ステップ3：標準棚割を作成する

1. 「**＋ 新規作成**」ボタンをクリックします。
2. モーダル画面が開きます。以下を入力してください。

| 項目 | 説明 | 例 |
|------|------|----|
| 棚割名 | 棚割を識別する名前 | `MEGA標準棚割（多段）` |
| 基準店舗 | 什器構成の基準とする店舗 | `東京中央店 (TK001)` |
| 適用開始日 | この棚割の適用開始日（任意） | `2024-01-15` |
| 適用終了日 | この棚割の適用終了日（任意） | `2024-03-31` |
| メモ・用途 | 自由記述（任意） | `2024年末年始用` |

3. 「**作成**」ボタンをクリックします。

[ここに新規作成モーダルのスクリーンショットを挿入]

#### ステップ4：棚ブロックを配置する（ドラッグ＆ドロップ）

1. 画面左側の「**棚ブロック**」パネルに、利用可能なブロック一覧が表示されます。
2. ブロックをマウスでドラッグし、右側の棚割キャンバス上にドロップします。
3. ブロックは自動的に空いている位置に左詰めで配置されます。
4. ブロック内の商品は、段（shelfIndex）と位置に従って自動展開されます。

> **注意**: スペースが不足している場合、「スペースが足りません。先に既存の商品を調整してください。」というメッセージが表示されます。不要なブロックを削除してから再度お試しください。

#### ブロックの削除

- 棚割キャンバス上部のブロック名の横にある「**×**」ボタンをクリックします。
- 確認ダイアログ「このブロックを削除してもよろしいですか？」で「OK」を選択します。
- ブロックとその範囲内の商品が削除されます。

#### 棚割のクリア

- 画面右上の「**全てクリア**」ボタンで、配置されたブロック・商品をすべて削除できます。
- 確認ダイアログ「この標準棚割をクリアしますか？配置された商品がすべて削除されます。」で「OK」を選択します。

#### 棚割の複製

- 標準棚割一覧テーブルの「**複製**」ボタンをクリックします。
- 元の棚割のブロック・商品配置がコピーされた状態で新規作成モーダルが開きます。
- 「📋「（元の棚割名）」のブロック・商品配置をコピーします」という案内が表示されます。

#### 棚割の削除

- 一覧テーブルの「**削除**」ボタンをクリックします。
- 確認ダイアログ「この標準棚割を削除しますか？」で「OK」を選択します。

### 分析モードの使い方

1. 棚割エディタ画面右上の「**📊 分析モード**」チェックボックスをオンにします。
2. 表示するメトリクスをドロップダウンから選択します。

| 選択肢 | 内容 |
|--------|------|
| 売上金額 | 各商品の売上金額をヒートマップで表示 |
| 粗利 | 各商品の粗利をヒートマップで表示 |
| 売上数量 | 各商品の売上数量をヒートマップで表示 |
| 客数 | 各商品の客数をヒートマップで表示 |

- 色の濃い商品ほど指標の値が大きいことを意味します。
- 各商品セルの右上に数値バッジが表示されます。

[ここに分析モード（ヒートマップ表示）のスクリーンショットを挿入]

### 棚割一覧テーブルの注意事項

- 「**⚠️ 重複**」バッジ: 他の棚割と適用期間が重複していることを示します。期間を調整してください。

## 5. 個店棚割管理

> **サイドバー** → 🏬 **個店棚割管理**

### この画面でできること

- 標準棚割をベースに、複数店舗の棚割を一括自動生成する
- 全店舗の棚割ステータスを一覧で確認する
- 個別店舗の棚割編集画面に遷移する

### 画面上部のステータスサマリー

画面上部にタブ切り替えの下に、以下のサマリーが常時表示されます。

| 項目 | 説明 |
|------|------|
| 全店舗 | 登録されている店舗の総数 |
| 生成済み | 棚割が正常に生成された店舗数（緑色） |
| 警告あり | 棚割は生成されたが警告がある店舗数（黄色） |
| 未生成 | まだ棚割が作成されていない店舗数 |

### タブ①: 🚀 自動棚割提案（一括生成）

このタブでは、1つの標準棚割を複数の店舗に一括適用できます。

#### 操作手順

**ステップ1：処理対象FMTを選択**

1. 「**① 処理対象FMT**」ドロップダウンからFMTを選択します。
2. 画面上のFMTカードをクリックしても選択できます。
   - カードには各FMTの店舗数と標準棚割の有無が表示されます。
   - 「**標準棚割あり**」（緑）/ 「**未設定**」（黄）で状態を確認できます。

**ステップ2：標準棚割を選択**

1. 「**② 標準棚割を選択**」セクションに、選択したFMTの標準棚割一覧が表示されます。
2. ラジオボタンで適用したい棚割を選択します。
3. 各棚割には以下の情報が表示されます。
   - 📅 適用期間
   - 💬 メモ・用途
   - 商品数
   - 什器タイプ

> 「このFMTの標準棚割がありません。先に「FMT標準棚割管理」で作成してください。」と表示される場合は、📋 FMT標準棚割画面で先に標準棚割を作成してください。

**ステップ3：対象店舗を選択**

1. 「**③ 対象店舗を選択**」セクションに店舗一覧テーブルが表示されます。
2. チェックボックスで生成対象の店舗を選択します。
3. 「**全選択**」「**選択解除**」ボタンで一括操作できます。
4. 各店舗の棚割ステータスが右列に表示されます。
   - ✓ 生成済み / 🔄 同期済み / ⚠️ 警告 / − 未生成

**ステップ4：一括生成を実行**

1. 「**🚀 自動棚割提案を作成**」ボタンをクリックします。
2. 確認ダイアログ「「（棚割名）」を X 店舗に対して一括生成しますか？」で「OK」を選択します。
3. プログレスバーで進捗が表示されます。
4. 完了後、生成結果テーブルに結果が表示されます。
   - **完了**: 正常に生成された店舗（緑バッジ）
   - **警告あり**: 生成されたが調整が入った店舗（黄バッジ）— 「**詳細**」ボタンで確認
   - **エラー**: 生成に失敗した店舗（赤バッジ）

[ここに一括生成画面と結果テーブルのスクリーンショットを挿入]

### 自動調整ルール

一括生成時、各店舗の什器幅と標準棚割の幅が異なる場合、以下のルールで自動調整されます。

| ルール | 条件 | 動作 |
|--------|------|------|
| **ルールA: カット** | 店舗棚 < 標準棚の場合 | 売上ランク下位から、フェイス削減→商品カットの順で縮小 |
| **ルールB: 拡張** | 店舗棚 > 標準棚の場合 | 売上ランク上位10商品のフェイスを2倍→1.5倍→余白で拡大 |
| **ルールC: 同期** | 標準棚割の更新後 | 個店棚割詳細画面の「🔄 同期」ボタンで最新状態に更新 |

### タブ②: ✏️ 個店編集

このタブでは、全店舗の一覧と棚割ステータスを確認し、各店舗の詳細編集画面に遷移できます。

#### 操作手順

1. 上部のフィルターで店舗を絞り込みます。
   - **店舗検索**: 店舗コードまたは店舗名で検索
   - **FMT**: FMTで絞り込み
   - **リージョン**: 地域で絞り込み
   - 「**クリア**」ボタンでフィルターをリセット
2. 店舗一覧テーブルの「**編集**」ボタンをクリックすると、個店棚割の詳細編集画面に遷移します。

## 6. 個店棚割詳細（個別編集）

> **個店棚割管理**の「✏️ 個店編集」タブ → 対象店舗の「**編集**」ボタン

### この画面でできること

- 個店の棚割を什器タイプごとに確認・編集する
- 商品のフェイス数を増減する
- 商品を削除する
- 商品パレットから新しい商品をドラッグ＆ドロップで追加する
- 標準棚割から最新状態に同期する
- 分析モードで売上・粗利のヒートマップを確認する

### 画面構成

- **左側**: 棚割ビジュアル（什器タイプごとにカード表示）
- **右側**: 商品パレット（ドラッグ元）

### 什器タイプの切り替え

画面上部のタブで切り替えます。

| タブ名 | 対象 |
|--------|------|
| 多段棚 | 多段棚・ゴンドラ |
| 平台   | 平台冷蔵・冷凍・エンドを含むすべての平台 |

各タブにはその什器タイプの総幅が表示されます。

### 棚割の自動生成

棚割がまだ作成されていない什器タイプについては、以下が表示されます。

1. 「この種類の棚割はまだ作成されていません」というメッセージが表示されます。
2. 標準棚割が複数ある場合は、ドロップダウンで選択できます。
3. 「**✨ 自動棚割提案を作成**」ボタンをクリックします。
4. 確認ダイアログ「標準棚割を基に、この種類の棚割を自動生成しますか？」で「OK」を選択します。

### 商品の編集

#### フェイス数の変更

1. 棚割ビジュアル上の商品セルに、「**-**」と「**+**」ボタンが表示されています。
2. 「**+**」をクリックするとフェイス数が1増えます。
3. 「**-**」をクリックするとフェイス数が1減ります（最小1）。

#### 商品の削除

1. 商品セルの「**×**」（赤いボタン）をクリックします。
2. その商品が棚割から削除され、残りの商品の位置が自動的に左詰めで再計算されます。

#### 新しい商品の追加（ドラッグ＆ドロップ）

1. 画面右側の「**商品パレット**」から追加したい商品を探します。
   - 検索ボックスで商品名・カテゴリで絞り込めます。
2. 商品をマウスでドラッグし、追加したい段（shelf-row）にドロップします。
3. 同じ商品を同じ段の最後尾にドロップすると、フェイス数が自動で+1されます。

> ⚠️ 段にスペースがない場合は「この段にはスペースがありません (残り: ○○mm, 商品: ○○mm)」というアラートが表示されます。

### 標準棚割との同期

1. 各什器タイプのカードヘッダーにある「**🔄 同期**」ボタンをクリックします。
2. 確認ダイアログ「標準棚割から最新状態に同期しますか？現在の個店編集は上書きされます。」で「OK」を選択します。
3. 標準棚割の最新内容で棚割が再生成されます。

> ⚠️ 同期すると、個店で行った手動の微調整（フェイス変更・商品追加など）はすべて上書きされます。

### 警告メッセージの確認

棚割に警告がある場合、カード上部に黄色の「**⚠️ 調整メッセージ**」セクションが表示されます。自動調整で発生した変更点（カットされた商品、フェイス数を変更した商品など）が一覧で確認できます。

### ステータスバッジの見方

| バッジ | 意味 |
|--------|------|
| **警告あり** （黄） | 自動生成時に調整が入った |
| **生成完了** （緑） | 正常に生成された |
| **同期済み** （紫） | 標準棚割から同期済み |

## 7. よくある質問・トラブルシューティング

### エラーメッセージと対処法

| メッセージ | 画面 | 原因と対処法 |
|-----------|------|-------------|
| 「⚠️ このFMTで什器配置済みの店舗がありません。先に「店舗棚尺マスタ」で什器を配置してください。」 | FMT標準棚割管理 | 選択したFMTの店舗に什器が配置されていません。「📐 店舗棚尺マスタ」画面で先に什器を配置してください。 |
| 「スペースが足りません。先に既存の商品を調整してください。」 | FMT標準棚割管理 | ブロックを配置するための空きスペースがありません。不要なブロックを削除してからやり直してください。 |
| 「この段にはスペースがありません (残り: ○○mm, 商品: ○○mm)」 | 個店棚割詳細 | 商品をドロップした段にスペースが不足しています。不要な商品を削除するか、フェイス数を減らしてください。 |

### よくある質問

**Q. 標準棚割を更新したら、すでに生成済みの個店棚割はどうなりますか？**

A. 自動的には反映されません。個店棚割詳細画面から「🔄 同期」ボタンを押すことで最新の標準棚割に更新されます。ただし、個店独自の編集内容は上書きされますのでご注意ください。

**Q. 棚割の段の表示順はどうなっていますか？**

A. 実際の什器と同じように、**画面の一番下が1段目**、上に向かって2段目、3段目…と増えます。


**Q. 一括生成で「警告あり」になった店舗はそのまま使って良いですか？**

A. 使用できます。「警告あり」は自動調整プロセスで何らかの変更（フェイス数の削減、商品のカットなど）が行われたことを示します。詳細を確認し、必要に応じて個店編集画面で微調整してください。

**Q. 複数の標準棚割パターンの適用期間が重複している場合はどうなりますか？**

A. 標準棚割一覧テーブルに「⚠️ 重複」バッジが表示されます。一括生成時は選択した1つの棚割のみが適用されるため、動作上の問題はありませんが、管理上は期間の整理を推奨します。
