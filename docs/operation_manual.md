## 1. はじめに

### システムの目的

本システムは、小売店舗の棚割（ゴンドラや平台冷蔵ケースなどの什器に、どの商品を・何フェイスで・どの段に配置するか）を効率的に管理するためのWebアプリケーションです。

### 基本的な業務フロー

```
① マスタデータ登録（商品・什器・店舗・店舗棚尺）
    ↓
② 棚ブロック作成（商品の組み合わせテンプレート）
    ↓
③ FMT標準棚割を作成（什器タイプ別）
    ↓
④ 個店棚割を自動生成（標準棚割をベースに各店舗の棚サイズに合わせて自動調整）
    ↓
⑤ 必要に応じて個店ごとに微調整
    ↓
⑥ 棚割指示書をPDF出力して店舗へ配布
```

## 2. 基本用語の解説

| 用語 | 説明 |
|------|------|
| **FMT** | 店舗フォーマット。店舗の業態分類のこと。本システムでは **MEGA**、**SuC**、**SMART**、**GO**、**FC** の5種類が定義されています。 |
| **什器（じゅうき）** | 店舗に設置する棚や陳列ケースのこと。「多段棚」「ゴンドラ」「平台冷蔵」「平台冷凍」「平台冷蔵エンド」「平台冷凍エンド」の種類があります。 |
| **尺（しゃく）** | 什器の幅を表す単位。1尺 = 300mm（30cm）です。例: 「3尺棚」= 幅900mmの棚。 |
| **棚ブロック** | 複数の商品をひとまとめにしたグループ単位。ブロック単位でドラッグ＆ドロップして棚割を組み立てます。 |
| **フェイス数** | 棚の正面から見た同一商品の並び数。フェイス数が多いほど、その商品の陳列面が広くなります。 |
| **段（だん）** | 什器の棚板の位置。画面上では **最下部が1段目**、上に行くほど段数が増えます（実際の什器と同じ向き）。 |
| **JAN** | JANコード（Japanese Article Number）。商品を識別する13桁のバーコード番号です。商品タイルにマウスを置くとツールチップで確認できます。 |
| **標準棚割** | FMTごとに定めた「お手本」となる棚割パターン。基準店舗の什器構成をもとに作成します。 |
| **個店棚割** | 各店舗固有の棚割。標準棚割をベースに、店舗の棚幅に合わせて自動調整されたものです。 |
| **リージョン** | 地域を表すグループ（北海道、東北、関東、中部、近畿、中国・四国、九州、全地域）。 |
| **売上ランク** | 商品ごとの売上順位。1が最高ランクです。自動調整時にどの商品を優先するかの判断に使用されます。 |
| **カテゴリ色分け** | 商品のカテゴリに応じて自動的に色を割り当てる機能。30色のパレットで一目でカテゴリを識別できます。 |

## 4. FMT標準棚割管理

> **サイドバー** → **FMT標準棚割**

### この画面でできること

- FMTごとの標準的な棚割パターンを作成・編集する
- 棚ブロックをドラッグ＆ドロップで配置する
- 既存の棚割を複製して新しいパターンを作る
- 分析モードで売上・粗利などのヒートマップを確認する
- カテゴリ色分けで商品グループを視覚的に把握する

### 操作手順

#### ステップ1：FMTと基準店舗を選択する

1. 画面上部の「**FMT選択**」ドロップダウンから対象のFMT（MEGA、SuC など）を選択します。
2. 「**基準店舗（什器配置済み）**」ドロップダウンから店舗を選択します。
   - ここに表示されるのは、「店舗棚尺マスタ」で什器を配置済みの店舗のみです。

> 店舗が表示されない場合は、先に「店舗棚尺マスタ」画面で什器を配置してください。

#### ステップ2：什器タイプを選択する

FMTの下に表示されるタブで什器の種類を切り替えます。

| タブ名 | 対象什器 |
|--------|---------|
| 多段   | 多段棚・ゴンドラ |
| 平台冷蔵 | 平台冷蔵ケース |
| 平台冷蔵エンド | 平台冷蔵エンドケース |
| 平台冷凍 | 平台冷凍ケース |
| 平台冷凍エンド | 平台冷凍エンドケース |

#### ステップ3：標準棚割を作成する

1. 「**＋ 新規作成**」ボタンをクリックします。
2. モーダル画面が開きます。以下を入力してください。

| 項目 | 説明 | 例 |
|------|------|----|
| 棚割名 | 棚割を識別する名前 | `MEGA標準棚割（多段）` |
| 基準店舗 | 什器構成の基準とする店舗 | `筑豊田川店 (MEGA0001)` |
| 適用開始日 | この棚割の適用開始日（任意） | `2026-04-01` |
| 適用終了日 | この棚割の適用終了日（任意） | `2026-06-30` |
| メモ・用途 | 自由記述（任意） | `2026年春夏用` |

3. 「**作成**」ボタンをクリックします。

#### ステップ4：棚ブロックを配置する（ドラッグ＆ドロップ）

1. 画面左側の「**棚ブロック**」パネルに、利用可能なブロック一覧が表示されます。
2. ブロックをマウスでドラッグし、右側の棚割キャンバス上にドロップします。
3. ブロックは自動的に空いている位置に左詰めで配置されます。
4. ブロック内の商品は、段（shelfIndex）と位置に従って自動展開されます。

> **注意**: スペースが不足している場合、「スペースが足りません。先に既存の商品を調整してください。」というメッセージが表示されます。不要なブロックを削除してから再度お試しください。

#### ブロックの削除

- 棚割キャンバス上部のブロック名の横にある「**×**」ボタンをクリックします。
- 確認ダイアログで「OK」を選択します。

#### 棚割のクリア

- 画面右上の「**全てクリア**」ボタンで、配置されたブロック・商品をすべて削除できます。

#### 棚割の複製

- 標準棚割一覧テーブルの「**複製**」ボタンをクリックします。
- 元の棚割のブロック・商品配置がコピーされた状態で新規作成モーダルが開きます。

### 商品の色分け表示

商品タイルはカテゴリに応じて自動的に色分けされます（30色パレット）。同じカテゴリの商品は同じ色で表示されるため、カテゴリの分布を一目で確認できます。

### 商品ツールチップ

商品タイル上にマウスカーソルを置くと、以下の情報がツールチップで表示されます:
- **商品名**（太字）
- **JAN**: JANコード
- **フェイス数**: ×N
- **品種**: カテゴリ名（設定されている場合）

### 分析モードの使い方

1. 棚割エディタ画面右上の「**分析モード**」チェックボックスをオンにします。
2. 表示するメトリクスをドロップダウンから選択します。

| 選択肢 | 内容 |
|--------|------|
| 売上金額 | 各商品の売上金額をヒートマップで表示 |
| 粗利 | 各商品の粗利をヒートマップで表示 |
| 売上数量 | 各商品の売上数量をヒートマップで表示 |
| 客数 | 各商品の客数をヒートマップで表示 |

- 色の濃い商品ほど指標の値が大きいことを意味します。

## 5. 個店棚割管理

> **サイドバー** → **個店棚割管理**

### この画面でできること

- 標準棚割をベースに、複数店舗の棚割を一括自動生成する
- 全店舗の棚割ステータスを一覧で確認する
- 個別店舗の棚割編集画面に遷移する

### 画面上部のステータスサマリー

| 項目 | 説明 |
|------|------|
| 全店舗 | 登録されている店舗の総数 |
| 生成済み | 棚割が正常に生成された店舗数（緑色） |
| 警告あり | 棚割は生成されたが警告がある店舗数（黄色） |
| 未生成 | まだ棚割が作成されていない店舗数 |

### タブ1: 自動棚割提案（一括生成）

#### 操作手順

**ステップ1：処理対象FMTを選択**

1. 「**処理対象FMT**」ドロップダウンからFMTを選択します。
2. FMTカードをクリックしても選択できます。
   - カードには各FMTの店舗数と標準棚割の有無が表示されます。

**ステップ2：標準棚割を選択**

1. 選択したFMTの標準棚割一覧がラジオボタンで表示されます。
2. 適用したい棚割を選択します。
3. 各棚割には適用期間・メモ・商品数・什器タイプが表示されます。

**ステップ3：対象店舗を選択**

1. 店舗一覧テーブルでチェックボックスにより生成対象を選択します。
2. 「**全選択**」「**選択解除**」ボタンで一括操作できます。

**ステップ4：一括生成を実行**

1. 「**自動棚割提案を作成**」ボタンをクリックします。
2. 確認ダイアログで「OK」を選択します。
3. プログレスバーで進捗が表示されます。
4. 完了後、生成結果テーブルに結果が表示されます。
   - **完了**: 正常に生成（緑バッジ）
   - **警告あり**: 生成されたが調整が入った（黄バッジ）
   - **エラー**: 生成に失敗（赤バッジ）

### 自動調整ルール

| ルール | 条件 | 動作 |
|--------|------|------|
| **ルールA: カット** | 店舗棚 < 標準棚の場合 | 売上ランク下位から、フェイス削減→商品カットの順で縮小 |
| **ルールB: 拡張** | 店舗棚 > 標準棚の場合 | 売上ランク上位10商品のフェイスを2倍→1.5倍→余白で拡大 |
| **ルールC: 同期** | 標準棚割の更新後 | 個店棚割詳細画面の「同期」ボタンで最新状態に更新 |

### タブ2: 個店編集

店舗検索（コード・名称）、FMT・リージョンフィルターで絞り込み、「**編集**」ボタンで詳細画面に遷移。

## 6. 個店棚割詳細（個別編集）

> **個店棚割管理**の「個店編集」タブ → 対象店舗の「**編集**」ボタン

### この画面でできること

- 個店の棚割を什器タイプごとに確認・編集する
- 商品のフェイス数を増減する（棚幅オーバーフローチェック付き）
- 商品を削除する
- 商品パレットから新しい商品をドラッグ＆ドロップで追加する
- 標準棚割から最新状態に同期する
- カテゴリ色分けで商品グループを確認する
- 商品ツールチップでJAN・フェイス数を確認する

### 什器タイプの切り替え

画面上部のタブで切り替えます。

| タブ名 | 対象 |
|--------|------|
| 多段棚 | 多段棚・ゴンドラ |
| 平台   | 平台冷蔵・冷凍・エンドを含むすべての平台 |

### 商品の編集

#### フェイス数の変更

1. 棚割ビジュアル上の商品セルに、「**-**」と「**+**」ボタンが表示されています。
2. 「**+**」をクリックするとフェイス数が1増えます。
   - 棚幅を超える場合は「スペースが不足しています（残り: XXmm）」というアラートが表示され、操作がブロックされます。
3. 「**-**」をクリックするとフェイス数が1減ります（最小1）。

#### 商品の削除

1. 商品セルの「**×**」（赤いボタン）をクリックします。
2. 残りの商品の位置が自動的に左詰めで再計算されます。

#### 新しい商品の追加（ドラッグ＆ドロップ）

1. 画面右側の「**商品パレット**」から追加したい商品を探します。
   - 検索ボックスで商品名・カテゴリで絞り込めます。
2. 商品をマウスでドラッグし、追加したい段にドロップします。
3. 同じ商品を同じ段にドロップすると、フェイス数が自動で+1されます。

> 段にスペースがない場合は「この段にはスペースがありません」というアラートが表示されます。

### 標準棚割との同期

1. 各什器タイプのカードヘッダーにある「**同期**」ボタンをクリックします。
2. 確認ダイアログで「OK」を選択します。
3. 標準棚割の最新内容で棚割が再生成されます。

> 同期すると、個店で行った手動の微調整はすべて上書きされます。

### 商品の色分けとツールチップ

- 商品タイルはカテゴリに応じて自動色分けされます。
- マウスホバーで商品名・JAN・フェイス数・品種のツールチップが表示されます。
- 見切れて読めない商品名もツールチップで確認できます。

## 7. 棚割指示書作成（PDF出力）

> **サイドバー** → **棚割指示書**

### この画面でできること

- 店舗ごとの棚割指示書をプレビューする
- 注釈（変更点・注意事項・補足）を追加する
- ブラウザの印刷機能を利用してPDF出力する

### 操作手順

#### ステップ1：対象店舗を選択する

1. 「**店舗名・コード検索**」テキストボックスで絞り込みます。
2. 「**FMT**」ボタン群で絞り込めます。
3. 「**対象店舗**」ドロップダウンから店舗を選択します。

#### ステップ2：指示書の設定

1. 「**指示書タイトル**」を編集します（デフォルト: 「棚割変更指示書」）。
2. 「**適用日**」を設定します。

#### ステップ3：注釈・コメントを追加する（任意）

| カテゴリ | 色 | 用途 |
|---------|-----|------|
| 変更点 | 青色 | 前回からの変更内容を記載 |
| 注意事項 | 黄色 | 施工時の注意点を記載 |
| 補足 | グレー | その他の補足情報を記載 |

1. カテゴリを選択し、内容を入力して「**＋ 追加**」をクリック。
2. 注釈はPDF出力時にも印刷されます。

#### ステップ4：PDF出力

1. 画面右上の「**PDF出力**」ボタンをクリックします。
2. ブラウザの印刷ダイアログが表示されます。

> **ブラウザ印刷設定の注意点**
> - 「**背景のグラフィック**」（または「背景の色と画像」）のチェックを **オン** にしてください。オフの場合、カテゴリ色分けやヒートマップが表示されません。
> - 送信先を「**PDFに保存**」に設定してください。

### 指示書の構成

| セクション | 内容 |
|-----------|------|
| ヘッダー | 指示書タイトル、作成日、適用日、店舗情報（コード・名称・FMT・地域） |
| 注釈・コメント | 追加した変更点・注意事項・補足 |
| 売り場レイアウト | 什器の2Dレイアウト表示（什器名、ブロック色分け表示付き） |
| 棚割ビジュアル | 什器タイプ別の棚割図（段番号・ブロック名・商品名・JAN・フェイス数を表示） |
| 棚ブロック詳細 | ブロックごとの商品一覧テーブル（商品名・フェイス数・段番号） |
| 商品リスト | 什器タイプ別の全商品一覧テーブル |

## 8. よくある質問・トラブルシューティング

### エラーメッセージと対処法

| メッセージ | 画面 | 原因と対処法 |
|-----------|------|-------------|
| 「このFMTで什器配置済みの店舗がありません。先に「店舗棚尺マスタ」で什器を配置してください。」 | FMT標準棚割管理 | 選択したFMTの店舗に什器が配置されていません。「店舗棚尺マスタ」画面で先に什器を配置してください。 |
| 「スペースが足りません。先に既存の商品を調整してください。」 | FMT標準棚割管理 | ブロックを配置するための空きスペースがありません。不要なブロックを削除してからやり直してください。 |
| 「スペースが不足しています（残り: XXmm）」 | 個店棚割詳細 | フェイス数を増やすためのスペースが棚段にありません。不要な商品を削除するか、他の商品のフェイス数を減らしてください。 |
| 「この段にはスペースがありません (残り: XXmm, 商品: XXmm)」 | 個店棚割詳細 | 商品をドロップした段にスペースが不足しています。 |
| 「このFMTの標準棚割がありません。先に「FMT標準棚割管理」で作成してください。」 | 個店棚割管理 | 選択したFMTの標準棚割がまだ作成されていません。 |
| 「標準棚割から最新状態に同期しますか？現在の個店編集は上書きされます。」 | 個店棚割詳細 | 確認ダイアログです。「OK」で手動編集が上書きされます。 |
| 「対応する標準棚割が見つかりません」 | 個店棚割詳細 | この什器タイプに対応する標準棚割が存在しません。 |
| 「棚ブロックがありません」 | FMT標準棚割管理 | 対応する棚ブロックが未登録です。「棚ブロック管理」で先に作成してください。 |

### よくある質問

**Q. 標準棚割を更新したら、すでに生成済みの個店棚割はどうなりますか？**

A. 自動的には反映されません。個店棚割詳細画面から「同期」ボタンを押すことで最新の標準棚割に更新されます。ただし、個店独自の編集内容は上書きされますのでご注意ください。

**Q. 棚割の段の表示順はどうなっていますか？**

A. 実際の什器と同じように、**画面の一番下が1段目**、上に向かって2段目、3段目…と増えます。

**Q. PDF出力で色がつかないのですが？**

A. ブラウザの印刷設定で「**背景のグラフィック**」（Chrome）または「**背景の色と画像を印刷**」（Firefox/Safari）をオンにしてください。

**Q. 一括生成で「警告あり」になった店舗はそのまま使って良いですか？**

A. 使用できます。「警告あり」は自動調整で何らかの変更（フェイス数の削減、商品のカットなど）が行われたことを示します。詳細を確認し、必要に応じて微調整してください。

**Q. 商品名が見切れて読めません。**

A. 商品タイルにマウスカーソルを置くと、ツールチップで商品名・JAN・フェイス数・品種が表示されます。この機能は棚ブロック管理・個店棚割・棚割指示書・分析ビジュアライザ・FMT標準棚割の全5画面で利用できます。

**Q. 商品の色は何を表していますか？**

A. 商品タイルの色はカテゴリを表しています。同じカテゴリの商品は同じ色で表示され、30色のパレットが使用されています。カテゴリの分布を視覚的に確認するのに役立ちます。

**Q. フェイス数を増やそうとしたら「スペースが不足しています」と出ました。**

A. 棚段の残りスペースが商品幅に満たない場合に表示されます。他の商品のフェイス数を減らすか、不要な商品を削除してスペースを確保してください。

**Q. 単位の「mm」と「尺」の関係は？**

A. 1尺 = 300mm（30cm）です。システムでは内部的にmm単位で管理しており、画面では「1200mm (4.0尺)」のように両方の単位で表示されます。什器の幅入力ではmm/尺を切り替えて入力できます。
